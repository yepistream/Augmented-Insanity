# Implementation Status

### **"does X work right now?"**

Updated whenever a
landed change moves a row across statuses.

Status legend:

- **Working** -- production-quality. Used in the verified samples.
- **WIP** -- functionally present, but with known caveats; see the linked
  reference for the specific limitations.
- **Planned** -- not implemented yet, but committed to ship.
- **Known bug** -- behaves wrong; tracked separately in
  [Known Issues](Known-Issues.md).
- **Minor bug** -- cosmetic or low-impact; tracked in
  [Known Issues](Known-Issues.md).

| Item | Status | Notes |
|------|--------|-------|
| Module loader | Working | Topological dependency resolution, missing-dep skip with clear log. |
| Host API 0.1.0 | Working | `set_locate_space_relation`, `get_vm`, `get_context`. |
| Host API 0.2.0 | Working | Adds `register_hand_tracker`, frame broker, `get_module_data_dir`. Forward-compat policy in place. |
| Camera-frame broker | Working | Single producer (currently arcore-headpose), arbitrary subscribers, synchronous fan-out on publisher's thread. |
| IPC hook dispatch | Working | Generated by `proto.py`; covers session, frame, swapchain, locate-space, locate-views, semantic-ids, device-get-tracked-pose. |
| Stub-xdev factory | WIP | Functional, but installs post-system-create; needs refactor (see [Stub Xdev Factory](Stub-Xdev-Factory.md) and [Roadmap](Roadmap.md)). |
| ARCore head-pose module | Working | 6DoF in helloxr.opengles and the Godot OpenXR Hand Tracking demo. See [ARCore Module Reference](ARCore-Module-Reference.md). |
| Mercury hand-tracking (ARCore camera) | WIP | Joints flow at ~10 Hz; first-init briefly competes with ARCore VIO; calibration via JSON only (no UI). See [Mercury Module Reference](Mercury-Module-Reference.md). |
| Mercury hand-tracking (USB camera) | Planned | Not implemented. Tracked in [Roadmap](Roadmap.md). |
| `ht_ctrl_emu` integration | Planned | Hand-to-controller pose emulation upstream code is present; not wired into a module yet. |
| Per-module calibration activity | Planned | Module-bundled ONNX intrinsics fallback works; no calibration UI yet. See [Calibration Activity](Calibration-Activity.md). |
| Module-provided Activities (DEX-loading) | Planned | Most likely in 0.2.0; Hopefully in 0.1.0. Lets module developers ship arbitrary Compose/View settings UI without a second APK. |
| Multi-process module isolation | Planned | All modules currently load into the runtime service process. |
| Settings UI for modules | Planned | Depends on module-provided Activities. |
| 11x lifecycle hook duplication | Known bug | Pre-existing; doesn't block functionality but multiplies IPC overhead. See [Known Issues](Known-Issues.md). |
| Misleading `In roles: <none>` log | Minor bug | Cosmetic; resolves when stub-xdev refactor lands. See [Known Issues](Known-Issues.md). |

## When something here is wrong

- **A row says "Working" but I observed otherwise**: file a bug
  ([bug report template](https://github.com/yepistream/Augmented-Insanity/issues/new?template=bug-report.md)).
- **A row is missing**: open a PR adding it.
- **A row says "Planned" and you want it sooner**: open an issue
  describing your use case so I can gawk at it.
